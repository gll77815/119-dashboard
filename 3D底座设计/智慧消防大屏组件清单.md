## 一、全局框架组件

### 组件 1 · 顶部状态栏（Zone T）

- **定位**：始终可见（48px 固定高），是身份与模式的视觉锚点。
- **功能**：展示建筑标识、当前模式、系统时间、值班人、设备在线率、外部气象；报警态压缩为极简带秒级计时。
- **展示方式**：
  - 左：建筑名称 + BIM 版本号（小字辅文）
  - 中：模式标识徽章（巡航绿色低亮 / 验证黄色 / 事件黄脉冲 / 报警红色脉冲带）
  - 右：时间 + 值班人 + 全楼在线率（数字 + 绿色细线进度）+ 温湿度
  - 报警态额外：报警计时 `00:03:42` / 起火位置 / 消防队 ETA
- **跨模式行为**：颜色与高度统一 48px；报警态触发**红色全屏边框脉冲光带**包裹整屏。
- **数据源**：BIM 元数据、值班排班系统、IoT 在线状态、气象 API。

### 组件 2 · 底部事件时间轴（Zone B）

- **定位**：水平时间轴（64px 固定高），既是实时操作台，也是事后复盘依据。
- **功能**：记录从触发到关闭的全部事件节点，支持向右实时延伸、历史回放、节点详情浮窗、人工备注。
- **展示方式**：
  - 节点色彩编码：橙=传感器触发 / 蓝=AI 判定 / 白=人工操作 / 灰=系统自动 / 红绿=关键里程碑
  - 节点形态：圆点 + 时间戳 + 简短文案；hover 浮出详情
  - 复盘态：底部进度条 + 播放/暂停/速率，节点可挂"批注气泡"
- **跨模式行为**：巡航仅显示 24h 正常节点；事件/报警时实时滚动；办公复盘可拖拽回放。

### 组件 3 · 模式状态机控制器

- **定位**：系统的"操作系统控制层"，决定屏幕进入哪种状态。
- **功能**：自动切换规则引擎（信号 + AI 置信度阈值）、人工干预（带原因选择）、越级跳转（已确认火情直入报警）、防抖（10s 内同源不重复触发）、报警下双重确认。
- **展示方式**：
  - 显式控件：顶栏右侧"模式切换器"按钮（常态折叠，点击展开 8 模式雷达图）
  - 隐式逻辑：后台规则引擎，用户无感
  - 切换日志：点击模式徽章可展开"为什么进入此模式"溯源面板
- **数据源**：传感器流、AI 置信度、人工事件、定时器。

### 组件 4 · 右侧指挥抽屉（Right-Drawer）

- **定位**：贯穿所有模式的通用信息层，落实"零页面跳转"。
- **功能**：承载模式专属内容；支持半开（详情）/ 全开（操作）/ 锁定全开（熔断）三态。
- **展示方式**：
  - 尺寸：0 / 220 / 420 px 三档，从右侧 200ms ease-out 滑入
  - 内部结构：顶部固定操作区 + 中部滚动详情区 + 底部行动按钮
  - 面包屑：抽屉内浏览路径，可回退
  - 顶部分屏切换器：报警态有"应急策略 / 通讯 / 资源"3 屏
- **跨模式行为**：
  - 巡航/验证：关闭或半开 AI 判定详情
  - 事件：半开多模态判定 + 视频 + 升级/排除
  - 报警：**锁定全开 420px**，不可关闭
  - 办公：按需展开异常指标命题卡 + 工单生成

### 组件 5 · 模式切换动画层

- **定位**：跨模式过渡的视觉中介，保证值班员"不会错过"关键切换。
- **功能**：根据来源/目标模式，播放对应过渡动画。
- **展示方式**：
  - 巡航 → 验证：无感（仅 Zone L 局部更新）
  - 验证 → 事件：Zone C 轻微放大 + 摄像头视图渐入
  - 事件 → 报警：**全屏黑色遮罩 0.5s → 报警界面闪出 + 红色边框脉冲**
  - 报警 → 巡航：红色边框渐退 + 3D 慢速复原 + Zone L 渐显

## 二、核心视图组件

### 组件 6 · 多层透明 3D 建筑看板

- **定位**：屏幕中央主视图，"楼宇安全认知接口"的核心载体。
- **功能**：8 个图层按需叠加，自动随模式切换默认图层与视角；支持点击下钻、手势旋转/缩放、BIM 属性弹出。
- **展示方式**：
  - 图层 L0–L8：基础白模 / IoT 设备点 / 摄像头视锥 / 人员热力 / 烟气推演 / 逃生路径 / 危险品 / 水管网 / 手绘进攻线
  - 状态光效：正常=绿色呼吸光晕；异常=淡黄脉冲；起火点=红色火焰图标 + 扩散圆 + 放射光圈
  - 视角策略：巡航慢速自转 3°/s；事件飞向楼层；报警锁定起火点不可旋转（5s 解锁窗口）
  - 右上角"图层控制面板"：每图层一个开关 + "模式推荐"标识
- **跨模式行为**：见组件矩阵；办公模式缩略至 Zone C 左半 300px。
- **数据源**：BIM 模型、IoT 台账、视频管理系统、Wi-Fi 探针、AI 烟气推演引擎。

### 组件 7 · 楼层平面图下钻视图

- **定位**：3D 看板的二级视图，承接楼层级别的精细操作。
- **功能**：双击 3D 楼层后展开 2D 平面图，显示设备分布、报警点、摄像头位置、疏散路径。
- **展示方式**：模态弹层覆盖 Zone C 80% 区域；右上角"返回 3D"按钮；可叠加同样的 L1–L8 图层。

### 组件 8 · 视频监控墙

- **定位**：常态视频汇聚视图（来自打印版"视频监控 11/12"模块）。
- **功能**：多路实时视频流缩略图，支持全屏、轮巡、画质切换、告警关联高亮。
- **展示方式**：
  - 网格：3×4 或 4×3 缩略图，每路含位置标签 + REC 红点 + 在线状态
  - 异常路：标 ⚠ 黄色边框；离线路：NO SIGNAL 占位
  - 顶部：「全屏 / 轮巡 / 在线 11 / 离线 1 / 告警关联 2 / 画质 2K·H.265」工具条
- **数据源**：视频管理系统（VMS）。

### 组件 9 · 视频 AI 判定融合视图

- **定位**：事件/报警模式下的视频主视图，叠加 AI 识别结果。
- **功能**：自动调取最近摄像头 → 视觉大模型连续 5 帧分析 → 输出判定结论；支持人工"AI 判定错误"反馈进入误报学习。
- **展示方式**：
  - 主画面：实时流（占 Zone C 60%）
  - 叠加层：检测框 + 置信度进度条 + 判定标签卡（如 `AI判定：低风险（85%概率为误报）`）+ 关键词高亮
  - 副画面：同楼层相邻 2–3 路摄像头自动布局
  - 反馈按钮：「✓ 判定正确 / ✗ 判定错误」
- **跨模式行为**：巡航/验证待机；事件主视图；报警辅助小窗；办公模式可调取历史录像复盘。

## 三、数据指标组件

### 组件 10 · 实时警情监控列表

- **定位**：左上侧栏的警情滚动条（来自打印版"实时警情监控"模块）。
- **功能**：按等级（火警/预警/故障/信息）实时滚动告警条目，每条带处置入口。
- **展示方式**：
  - 顶部计数胶囊：`火警 1 · 预警 3 · 故障 2 · 信息 2`
  - 列表项：等级图标 + 设备位置 + 时间 + 当前值/阈值对比 + 「处置」按钮
  - 高亮：火警条整行红色背景脉冲
  - 顶部工具：实时刷新（5s）+ 筛选 + 静音切换
- **数据源**：消防联动主机、IoT 网关、AI 识别结果。

### 组件 11 · 顶栏 KPI 胶囊条

- **定位**：顶栏下沿的 8 个核心指标胶囊（打印版的"全局指标条"）。
- **功能**：一屏看完今日总体态势。
- **展示方式**：横向 8 个胶囊，含今日火警 / 活跃预警 / 视频在线 / 值班在岗 / AI 引擎 / 设备在线率 / 联动规则 / 高风险设备；每胶囊：图标 + 数字 + 标签。
- **跨模式行为**：报警态全部隐藏；办公态可点击下钻。

### 组件 12 · 消防安全健康度卡片

- **定位**：将全楼消防状态压缩为 0–100 分的综合评分。
- **功能**：四维加权（物 40% / 管理 30% / 环境 20% / 人 10%）；支持 360° 一键检测；输出 Top 3 失分项与改善建议。
- **展示方式**：
  - 巡航态（Zone L 顶部）：圆环主分 + 7 日趋势折线 + 与上次差值
  - 办公态（Zone C 核心）：四维度雷达图 + 历史趋势 + 「360° 一键检测」按钮
- **跨模式行为**：报警/指挥/推演时**完全隐藏**。

### 组件 13 · IoT 设备监测卡

- **定位**：设备资产维度的健康度面板（打印版"IoT 设备监测 3,482 台"模块）。
- **功能**：按类型分布、健康度分布、在线率、巡检完成率、本月维保、待整改的多维汇总。
- **展示方式**：
  - 顶部：总台数 + 在线率大字
  - 类型条：烟感/温感/手报/电气/水压/气感水平进度条
  - 健康度堆叠条：优良 78% / 一般 16% / 待修 6%
  - 底部 KPI：巡检完成率 + 本月维保次数 + 待整改条数

### 组件 14 · 视频监控状态卡

- **定位**：视频侧的统计卡片（与组件 8 对应的 KPI 视图）。
- **功能**：在线/离线/告警关联数 + 编码格式 + 在线率。
- **展示方式**：纯数字徽章组，可作为视频监控墙的工具条。

### 组件 15 · 动态风险榜单

- **定位**：Zone L 的"风险温度计"，AI 静默判定结果在此沉淀。
- **功能**：按区域/楼层显示风险指数排行；验证模式触发时数字悄然爬升 + 颜色由白转黄；点击展开 AI 判定依据。
- **展示方式**：
  - 列表项：区域名 + 风险分（0–100）+ 趋势箭头 + 颜色渐变背景
  - 微动效：分值变化时数字滚动 + 颜色补间，无弹窗无声音
  - 点击：抽屉展开「事件类型 / AI 置信度 / 历史同类 / 生成工单」

### 组件 16 · 指标追踪卡片（命题卡）

- **定位**：所有指标的标准展示单元——"不是仪表盘，是决策推动器"。
- **功能**：每张卡片必含四要素：事实 / 归因 / 影响 / 建议（带行动按钮）；附带时间标签、置信度、重要性、授权级别。
- **展示方式**：
  - 摘要态：一句话 + 关键数字 + 等级图标（用于巡航 Zone L 列表）
  - 展开态：完整四要素 + 「一键派单」等行动按钮（点击后抽屉内）
  - 对话态：可继续追问/下钻/推演的活动卡（语音交互场景）
- **跨模式行为**：巡航 3–5 张；报警**全部隐藏**；办公全量按优先级排序。

### 组件 17 · 态势分析面板

- **定位**：办公模式 / 底部驻留的多图分析区（打印版"态势分析 近 7 日"模块）。
- **功能**：告警趋势、设备类型分布、处置时效漏斗。
- **展示方式**：
  - 折线图：火灾/预警/故障三色叠加，X 轴日期
  - 柱状图：设备类型分布
  - 漏斗图：触发 → 判定 → 派单 → 处置 → 关闭

### 组件 18 · 消防资源面板

- **定位**：人 / 物 / 水的可调度资源全景（打印版"消防资源 3 站 · 42 栓"模块）。
- **功能**：微型消防站在岗、物资储备、消火栓分布、水源（市政/天然）。
- **展示方式**：
  - 三列卡片：每个微站显示在岗比 `6/8` + 名称 + 状态
  - 消火栓：总数 + 正常/故障分色徽章
  - 水源：图标 + 数量
- **联动**：报警态此面板进入抽屉第三屏并叠加 BIM 定位。

### 组件 19 · 业务联动面板

- **定位**：日常业务运转视图（打印版"业务联动 自动化 5 规则"模块）。
- **功能**：今日巡检进度 + 自动化规则数量 + 仿真演练入口 + 联动自动化入口。
- **展示方式**：胶囊式 KPI + 两个跳转入口卡（🎯 仿真演练 / ⚡ 联动自动化）。

### 组件 20 · 隐患整改追踪卡

- **定位**：右侧的"未完待办墙"，对接清零助手（打印版"隐患整改追踪"列表）。
- **功能**：列出待整改 / 整改中 / 已完成 / 逾期的具体条目，含截止日。
- **展示方式**：列表项：等级图标 + 隐患描述 + 状态标签 + 截止日期；点击进入工单抽屉。

### 组件 21 · 值班 & 排班卡

- **定位**：Zone L 底部的人员归属信息。
- **功能**：当班人员、下次班次时间、紧急联系人快捷呼叫。
- **展示方式**：头像 + 姓名/工号 + 班次时间条 + 一键呼叫按钮。

## 四、AI 智能组件

### 组件 22 · AI 智能研判面板

- **定位**：右侧的 AI 引擎运行状态板（打印版"AI 智能研判 4 引擎运行中"模块）。
- **功能**：展示当前在跑的 AI 引擎及今日识别量；列出最新 AI 识别事件。
- **展示方式**：
  - 顶部：4 引擎徽章（烟火检测 / 人员聚集 / 通道阻塞 / 风险预测）+ 今日数量 + 准确率
  - 列表：最新识别条目（时间 + 位置 + 描述 + 置信度），缩略帧可点击放大

### 组件 23 · 消防隐患"清零"助手

- **定位**：办公模式主区的主动治理大脑——后台跑、关键时刻提醒。
- **功能**：长期数据积累 → 误报模式识别 → 归因推理（结合 BIM 空间属性）→ 主动建议命题卡 → 整改效果追踪。
- **展示方式**：
  - 主界面四件套：误报热点地图（楼层热力）+ 高风险设备列表 + 建议命题卡列表 + 整改效果对比
  - 命题卡：事实（误报统计）+ 归因（位置特征）+ 建议（灵敏度/位移/更换）+ 预估成本
- **跨模式行为**：巡航/办公后台跑；验证态在事件匹配历史误报模式时主动提示"该设备历史误报率 83%"；事件/报警态暂停。

### 组件 24 · 合规预检模块

- **定位**：年检前的差距分析与整改追踪——把合规压力转为可执行清单。
- **功能**：自动差距分析（对标规范库）→ 整改计划生成（难度 × 时间矩阵）→ 进度追踪 → 预检报告生成（自然语言指令触发）。
- **展示方式**：
  - 年检倒计时大字 + 不符合项数量
  - 差距项列表：条款依据 + 当前状态 + 整改要求 + 责任人
  - 进度看板：待整改 / 整改中 / 待验收 / 已完成 四列泳道
  - 预检自评报告导出区

### 组件 25 · 语音交互模块（防卫士）

- **定位**：贯穿全屏的对话层，"对话是主干，可视化是画布"。
- **功能**：唤醒词激活 / 麦克风按钮；意图识别（查询/分析/指令/导航/推演）；下一步意图推断；视图联动（提到楼层 3D 自动飞过去）；报警态无需唤醒词。
- **展示方式**：
  - 静默态：屏幕角落小型麦克风图标 + 呼吸光
  - 激活态：底部弹出输入气泡 + 实时语音波形
  - 输出：屏幕浮出命题卡 + 同步语音朗读 + 3D 视图联动
  - 推荐追问：答完后下方浮出 1–2 个相关问题胶囊

### 组件 26 · AI 应急策略执行面板

- **功能**：AI 推荐 N 项联动动作，支持「一键全部执行」或「逐条审批」；每项标授权级别；防火门等 L4 预授权动作自动执行后打 ✓ 已执行标记。
- **展示方式**：
  - 顶部统计：`AI 建议执行以下 [5] 项联动动作`
  - 复选列表：☐ 切断电源 / ☐ 迫降电梯 / ☐ 开启风阀 / ☑ 关闭防火门（已自动）/ ☐ 启动广播
  - 居中**大红色**「一键全部执行」按钮
  - 右上角切换：「逐条确认」模式 toggle

### 组件 27 · 联动控制面板

- **定位**：可控子系统的统一操作面板（消防电源 / 电梯 / 排烟风阀 / 防火门 / 喷淋 / 疏散广播 / 门禁 / 摄像头）。
- **功能**：每项当前状态 + 目标动作按钮 + 授权级别标识；批量"一键执行 AI 策略"；操作实时进时间轴；5 秒撤销窗口（不可逆动作除外）。
- **展示方式**：
  - 表格：子系统 | 状态 | 动作按钮 | 授权 | 撤销倒计时
  - 颜色：可执行=蓝；执行中=黄；已完成=绿；不可逆=灰禁用
- **跨模式行为**：巡航/验证隐藏；事件预备态（灰色不可点）；报警展开/执行；指挥/推演只读。

### 组件 28 · 战时通讯面板

- **定位**：报警抽屉第二屏，外部协同入口。
- **功能**：消防队接警状态 + ETA、物业应急联系人一键呼叫、上报记录（自动生成事件报告草稿）。
- **展示方式**：
  - 顶部：消防队状态卡（接警时间 + ETA 倒计时）
  - 中部：联系人列表，每行带"呼叫"按钮
  - 底部：自动生成的事件报告草稿预览 + 「确认上报」按钮

## 八、设计建议（给后续组件设计的领单建议）

1. **优先级建议**：先做 1/2/3/4/6/16/26（构成"巡航 → 报警"主链路的最小可演示集），再补 12/13/15/22 的常态数据展示。
2. **复用粒度**：组件 16（命题卡）是跨场景复用最高的原子单元，建议抽成可由 props 驱动的标准组件，所有指标/AI 输出/抽屉详情都基于它扩展。
3. **状态契约**：每个组件设计时显式声明 8 模式下的可见性与展示态（参考第七章矩阵），避免后期再补漏。
4. **色彩语义**：全局统一——绿=正常 / 蓝=AI / 黄=验证·疑似 / 橙=预警 / 红=报警·火情 / 灰=离线·暂停。
5. **熔断特例**：组件 1/4/16/12/13/15/17/19/20/22/24/30 在报警态都需要给出"折叠/隐藏/锁定"分支稿，建议在 Figma 中独立维护"熔断变体"组。
6. **数据契约**：建议把数据来源整理为统一接口表（IoT 网关 / VMS / BIM / AI 推理 / 排班 / 工单 / 规范库），避免每个组件各自约定。
