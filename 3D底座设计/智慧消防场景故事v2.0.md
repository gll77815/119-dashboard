# 智慧消防大屏 · 场景故事与组件规范

> 版本 v0.2 | 在初稿基础上补全各模式屏幕内容、功能项、AI 配置；补全组件功能与状态规格；新增 5 个核心组件

---

## 文档导读

### 核心设计哲学

大屏不是"看数据的屏"，而是**楼宇安全认知接口**。它将分散在 IoT 传感器、BIM 模型、视频系统、消防联动系统中的数据，编排为一个决策者能在一个界面完成**看、问、办**的统一入口。

AI 的角色在不同模式中滑动变化：平时是**安全参谋**（L1–L2），战时是**执行代理**（L3–L4）。这种角色的动态切换，是 AI 驾驶舱区别于传统大屏的根本。

---

### 模式状态机

八种模式构成一个状态机，转换由系统事件自动触发，也可人工干预。

```
                        ┌──────────────────────────────┐
                        │          巡航模式             │  ← 默认态 / 无事件
                        └───────────┬──────────────────┘
                                    │ 检测到异常信号（低置信度）
                                    ▼
                        ┌──────────────────────────────┐
                        │          验证模式             │  ← AI 静默判定
                        └───────┬──────────┬───────────┘
                   AI判定:低风险 │          │ AI判定:疑似火情
                        ▼        │          ▼
               记录日志，返回    │   ┌──────────────────────────────┐
               巡航模式          │   │          事件模式             │  ← 多模态融合确认
                                 │   └───────┬──────────┬───────────┘
                                 │  确认误报 │          │ 视频AI确认火情
                                 │           ▼          ▼
                                 │       返回巡航  ┌──────────────────┐
                                 │                 │  报警模式/熔断   │  ← 最高优先级
                                 │                 └───┬──────┬───────┘
                                 │           需战术推演│      │ 需现场指挥
                                 │                    ▼       ▼
                                 │          ┌──────────┐ ┌──────────┐
                                 │          │推演/接管 │ │应急指挥  │
                                 │          └──────────┘ └──────────┘
                                 │                    │       │
                                 │              火情扑灭，解除警报
                                 │                    │       │
                                 └────────────────────▼───────▼
                                              巡航模式 + 触发事后复盘

[ 公共模式 ]  独立运行，接收报警模式联动信号后切换为紧急疏散态
[ 办公模式 ]  随时可叠加在巡航模式之上；接收到报警信号自动跳转报警模式
```

### 屏幕基础布局框架

所有模式共用同一套物理布局区域，各区域的**可见性与内容**随模式变化。

```
┌─────────────────────────────────────────────────────────┐
│  Zone T · 顶部状态栏（固定 48px，始终可见）              │
├──────────┬──────────────────────────────┬───────────────┤
│          │                              │               │
│  Zone L  │       Zone C                 │  Zone R       │
│  左侧    │       中央 3D 模型区          │  右侧         │
│  信息面板 │      （弹性宽度，核心视图）   │  控制/数据    │
│  240px   │                              │  面板 / 抽屉  │
│          │                              │  220px → 420px│
│          │                              │               │
├──────────┴──────────────────────────────┴───────────────┤
│  Zone B · 底部事件时间轴（固定 64px，始终可见）           │
└─────────────────────────────────────────────────────────┘
```

**熔断状态（Melt-off）特例：** Zone L 隐藏，Zone R 强制展开为全宽抽屉，Zone C 全屏无遮挡。

# 一、大屏看板模式

---

## 1. 巡航模式

**核心目标：极致降噪、虚警过滤、证明系统活着**

### 原始描述

> 绝对的暗色系底噪设计。平时保持视觉静默，证明系统"活着"。满屏的绿灯和正常的数据，是在证明这座建筑的防御系统处于随时可触发的待命状态。屏幕中央的 3D 建筑 FUI 模型静默运转，边缘分布着透明悬浮卡片，展示各楼层正常指标。

---

### 屏幕内容布局

**Zone T · 顶部状态栏**
- 建筑名称 + BIM 版本号
- 当前模式标识：`● CRUISE 巡航中`（绿色低亮度）
- 系统时间 + 值班人员姓名/工号
- 全楼设备在线率（数字+绿色细线进度）
- 天气 + 外部气象感知（温湿度）

**Zone L · 左侧信息面板（240px，透明卡片）**

| 卡片 | 内容 |
|------|------|
| 消防安全健康度 | 综合评分 0–100，当前分值与7日趋势折线 |
| 设备状态汇总 | 在线率、离线数、维护中数量 |
| 今日巡检完成率 | 已完成 / 计划总数，进度条 |
| 动态风险榜单 | 各楼层/区域风险指数排行，实时微调 |
| 值班 & 排班信息 | 当班人员、下次班次时间 |

**Zone C · 中央 3D 建筑模型**
- 全楼 3D 白模，慢速自动旋转（约3°/秒，可手势暂停）
- 各楼层显示绿色呼吸光晕（正常态标识）
- 设备层叠加：在线探测器显示为细小白点，覆盖密度可感知
- 楼层点击可展开该楼层平面图（全屏模式）
- 底部环形装饰：实时时间刻度 + 系统心跳波形（证明系统在运行）

**Zone R · 右侧面板（220px，静态信息）**
- 分楼层设备状态条（竖向列表）
- 本月隐患整改进度
- 下次维保计划（最近3条）
- AI 参谋今日简报入口（折叠，点击展开）

**Zone B · 底部事件时间轴**
- 显示最近 24 小时事件（全部已关闭/正常态）
- 无未处理事件时显示"过去24小时无告警"绿色标识

---

### 功能可用项

| 功能 | 交互方式 | 说明 |
|------|----------|------|
| 楼层下钻 | 点击 3D 楼层 | 展开该楼层平面图，显示设备分布 |
| 设备详情 | 点击设备图标 | 弹出设备卡：型号/安装时间/历史故障/预计寿命 |
| AI 简报查看 | 点击 Zone R 入口 | 今日 AI 安全参谋简报，3条命题卡 |
| 语音提问 | 唤醒词 / 麦克风按钮 | 向 AI 参谋提问任意安全问题 |
| 手动切换模式 | 顶部模式切换器 | 可切换至办公模式 |
| 历史回溯 | 底部时间轴拖拽 | 查看任意时刻的历史状态快照 |
| 报表导出 | Zone R 底部按钮 | 生成当前日/周/月报 |

---

### AI 能力配置

| 维度 | 配置 |
|------|------|
| 授权级别 | L1（展示+解释，不主动推送处置建议） |
| 主要认知模式 | C-1 异常发现与定性（后台运行，静默） |
| 主要交互模式 | I-1 主动汇报（简报），I-2 对话式探查（按需） |
| AI 主动推送 | 关闭（仅当清晨 07:00–08:00 推送今日安全简报） |
| 自动执行权限 | 无 |

---

### 进入 / 退出条件

**进入**：系统启动默认进入；报警解除后自动回归；人工选择。

**退出触发**：
- 任意传感器触发阈值 → 自动切换至**验证模式**
- 人工点击"切换至办公模式" → **办公模式**
- 接收到已确认火情信号 → 直接跳转**报警模式**（跳过中间模式）

## 2. 验证模式

**核心目标：AI 静默判定，隐患沉淀，减少人工误扰**

### 原始描述

> 监测追踪 → 发现事件 → AI静默判定 → 非火情记录日志。AI探针判断风险级别较低，大屏上并未弹窗报警。但系统后台自动生成了一条隐患记录，并通过左侧的"动态风险榜单"卡片，将该商户的风险指数悄悄上调。

---

### 屏幕内容布局

验证模式对用户几乎**不可见**——它不会打断值班员的巡航视图，只会在局部区域发生微小变化。

**Zone L · 左侧面板变化**
- **动态风险榜单**：对应区域的风险分数在数秒内悄然上升（无弹窗，无声音）
  - 变化方式：数字从 32 缓慢爬升至 47，颜色从白色渐变为淡黄色
  - 悬停可查看：「AI发现 1 条低风险事件，已静默记录」

**Zone C · 中央 3D 模型变化**
- 对应楼层 / 区域：正常绿色光晕 → 极淡黄色脉冲（2Hz，低亮度）
- 其余楼层不变
- 无弹窗、无动效放大、无任何打断

**Zone R · 右侧面板**
- 无变化，维持巡航状态
- 若值班员点击已上调的风险榜单条目，Zone R 展开：
  - 事件类型（如：电气异常谐波）
  - AI 判定置信度与依据
  - 历史同类事件记录
  - 「查看详情」→「生成隐患工单」入口

**后台动作（不显示在大屏，记录于系统日志）**
- 设备监测日志：记录触发时间、设备ID、原始数值
- 模型识别/判定结果日志：AI推理链、置信度
- 如同一区域在 24 小时内触发 3 次及以上 → 自动升级为事件模式

---

### 功能可用项

| 功能 | 交互方式 | 说明 |
|------|----------|------|
| 查看 AI 静默记录 | 点击动态风险榜单条目 | 展开 Zone R 抽屉，显示AI判定详情 |
| 生成隐患工单 | Zone R 内按钮 | 一键生成"清零助手"工单 |
| 忽略此次事件 | Zone R 内按钮 | 标记为"已确认无风险"，清除榜单高亮 |
| 手动升级事件 | Zone R 内按钮 | 人工判断需要进一步核查，切换至事件模式 |
| 追问 AI 依据 | 对话框 | "为什么这次没有报警？" |

---

### AI 能力配置

| 维度 | 配置 |
|------|------|
| 授权级别 | L1（静默记录）+ L2（当榜单被点击时，提供建议） |
| 主要认知模式 | C-1 异常发现与定性（核心），C-2 归因分析 |
| 主要交互模式 | I-3 归因下钻（按需），I-5 建议-审批（生成工单需审批） |
| 自动执行权限 | 仅记录日志和更新风险指数；不发出任何通知或执行任何动作 |

---

### 进入 / 退出条件

**进入**：任意传感器触发，AI 初步判定置信度 < 60%；或单传感器触发（多源未联动）。

**退出触发**：
- AI判定：误报/低风险，置信度 > 80% → **返回巡航模式**（风险榜单保留记录）
- 多传感器联动触发 / AI判定置信度提升 > 60% → **事件模式**
- 同区域24h内触发3次及以上 → **事件模式**（自动升级）
- 人工判断需核查 → **事件模式**

## 3. 事件模式

**核心目标：多模态虚警过滤，快速确认或排除，减少人跑腿**

### 原始描述

> 非确定性火情 → 探针报警 → 集成监测：摄像头+监测报警+判定说明 + 操控手段 → 接管模式。多模态虚警过滤（Multi-sensor Fusion Verification）：当一个烟感报警时，AI 自动调取最近的摄像头，通过视觉大模型识别是否有明火或浓烟。黑灯值守 Agent：确认真实火警时，系统瞬间切换至"熔断状态"。

---

### 屏幕内容布局

**Zone T · 顶部状态栏**
- 模式标识变为：`◆ EVENT 事件核查中`（黄色，脉冲）
- 新增：事件编号 + 发生时间 + 持续时长计时器

**Zone L · 左侧面板（内容切换）**

| 区域 | 内容 |
|------|------|
| 顶部：事件摘要卡 | 触发传感器数量、位置、类型；AI初步置信度 |
| 中部：多传感器融合状态 | 各传感器状态矩阵（烟感/温感/视频/电气），逐项打勾/叉 |
| 下部：AI 判定进度 | 正在分析...→ 判定结果卡（事实+归因+置信度） |
| 底部：快速操作区 | 「确认误报」/ 「升级为火情」/ 「呼叫支援」 |

**Zone C · 中央区域（3D模型 + 视频融合）**

**视图一：3D定位视图**（默认）
- 3D建筑模型自动旋转停止，视角飞向事件楼层
- 报警点：红色闪烁圆球标注（带放射性光圈）
- 覆盖摄像头视锥：蓝色半透明三角，显示当前摄像头观测范围
- 临近摄像头自动标注并高亮"AI正在分析"

**视图二：视频融合视图**（点击摄像头图标切换）
- 主画面：最近报警点摄像头画面（实时），全屏可切换
- AI分析叠加层：
  - 检测框（如：识别到水蒸气/抽烟/明火）
  - 判定标签：`AI判定：低风险（85%概率为误报）`或`AI判定：疑似火情（72%置信度）`
  - 依据关键词高亮（如：温度曲线正常、无明火特征）
- 副画面：同楼层相邻摄像头（2–3路，自动布局）

**Zone R · 右侧半开抽屉（事件详情）**
- 事件时间轴（当前事件从触发至今的所有节点）
- 传感器原始数值曲线（触发前后60秒）
- 历史同类事件参考（"2024年3月同位置类似触发，结论：误报，原因：加湿器"）
- 消防预案快速查阅入口

**Zone B · 底部事件时间轴**
- 从事件触发开始实时滚动记录每个节点

| 视频实时查看 | Zone C 视图切换 | 多路摄像头实时画面 |

| 主要交互模式 | I-2 对话式探查、I-3 归因下钻、I-5 建议-审批 |

### 进入 / 退出条件

**进入**：多传感器联动触发；或验证模式升级；或人工强制触发。

**退出触发**：
- 人工确认误报 → **返回巡航模式**
- 视频AI + 多传感器联动确认火情 → **报警模式/熔断**
- AI判定置信度 > 85% 火情 + 2路以上传感器 → **系统建议升级，值班员确认后进入报警模式**

## 4. 报警模式

**核心目标：信息最大化集中，指令最小化操作，秒级响应**

### 原始描述

> 大屏正式进入"熔断（Melt-off）状态"。原本两侧常规的悬浮卡片瞬间隐藏。中央3D GIS核心模型绝对无遮挡，视角自动拉近并锁定起火点。从右侧滑出右侧抽屉（Right-Drawer）指挥面板。AI联动执行：切断非消防电源，迫降电梯，开启排风阀。值班员只需点击"一键执行"。

---

### 屏幕内容布局

> **熔断机制生效：Zone L 完全隐藏，Zone C 扩展至全屏，Zone R 强制展开为全宽指挥抽屉（420px），Zone T 简化为极简状态栏。**

**Zone T · 极简状态栏**

## 组件 2：消防隐患"清零"助手 — 设备误报分析

**简介：** 后台运行的主动治理系统，将"被动消防"转为"主动消防"。

### 功能说明

1. **长期数据积累**：持续记录每个探测器的报警历史（含误报/真警）、环境上下文（温度/湿度/时段）
2. **误报模式识别**：AI识别高频误报探测器的行为模式（如某探测器在08:30–09:30每天误报1次）
3. **归因推理**：对比BIM空间属性（如探测器附近有厨房、加湿器、吸烟区），生成原因推断
4. **主动建议**：生成命题卡，告知管理者"建议调整X探测器灵敏度或移位"
5. **效果追踪**：整改执行后，持续监测该探测器的误报率变化，生成"治理效果报告"

### 各模式下的状态

| 模式 | 状态 | 主要行为 |
|------|------|----------|
| 巡航/办公 | 后台运行 | 持续分析历史数据，生成建议命题卡 |
| 验证 | 触发关联 | 当前触发事件匹配历史误报模式时，主动提示"该设备历史误报率83%" |
| 事件 | 输入依据 | 为多模态判定提供该设备历史行为参考 |
| 报警及以后 | 暂停 | 全部资源让出，专注当前火情处置 |

### 主界面（在办公模式 Zone C 展开）

- **误报热点地图**：按楼层/区域显示误报频次热力图（过去30天）
- **高风险设备列表**：最近30天误报率 > 30% 的设备，排序
- **建议命题卡列表**：每个建议含：事实（误报统计）+ 归因（位置特征）+ 建议（灵敏度调整/位移/更换）+ 预估整改成本
- **整改效果追踪**：已执行建议的效果对比（整改前/后误报率）

### 数据源

- 探测器历史报警记录（含AI判定结果）
- BIM空间属性（探测器安装位置附近的功能区分类）
- 环境传感器（温湿度、时段）
- 整改工单系统（执行记录）

## 组件 4：视频 AI 隐患判定

**简介：** 事件触发时，自动调取摄像头通过视觉大模型判定是否为真实火情，解决烟感频繁误报问题。

### 功能说明

**触发机制**：传感器报警 → 自动匹配最近摄像头（BIM空间关联）→ 拉取实时视频流

**判定流程**：
1. 视觉大模型分析：识别画面中是否有明火/浓烟/高温光晕
2. 历史帧对比：与同摄像头历史帧对比（检测异常变化）
3. 多帧置信度积累：连续5帧均触发 → 置信度累加
4. 输出判定结论：`AI判定：低风险（85%概率为误报）原因：检测到水蒸气，无明火特征`

**界面展示**：
- 主画面：实时视频流
- 叠加层：
  - 检测框（标注识别对象）
  - 置信度进度条（实时更新）
  - 判定关键词高亮
- 副画面：同楼层2–3路相邻摄像头

### 各模式下的状态

| 模式 | 状态 |
|------|------|
| 巡航/验证 | 待机；如有AI静默标注的区域，定期抽帧分析 |
| 事件 | 核心功能激活，多摄像头协同分析，实时显示在Zone C |
| 报警 | 辅助监控（小画中画），主要辅助确认火情位置 |
| 办公 | 复盘模式：调取历史录像 + AI重新分析，生成事后报告 |

### 误报学习机制

当人工确认"AI判定错误"时：
- 记录该次判定的输入帧和错误结论
- 自动提交至模型优化队列
- 相似场景（同区域/同时段/同环境条件）下的判定阈值自动微调

## 组件 5：指标追踪卡片

**简介：** 不是传统的数字仪表盘，而是以"命题"为单位的信息展示单元。每张卡片都是一个完整的决策推动器。

### 命题卡结构规范

每张指标卡片包含以下四个必填要素（缺一不可）：

| 要素 | 内容要求 | 示例 |
|------|----------|------|
| 事实（What） | 客观数值陈述，含时间维度 | "3楼东北区 S3-NE-047 已离线 26 小时" |

| 建议（Now What） | 可直接执行的动作，带行动入口 | "今日17:00前核查 + 采购替换 ← [一键派单]" |

**附加属性**：
- 时间标签（历史/当下/预测/反事实）
- 置信度（87%）
- 重要性等级（高/中/低）
- 授权级别标识（L3 · 需审批）

### 卡片三种展示态

| 状态 | 适用场景 | 内容 |
|------|----------|------|

| 展开态 | 点击后详情查看 | 完整四要素 + 行动按钮 |
| 对话态 | AI参谋追问 | 可继续提问、下钻、推演的活动卡片 |

### 各模式下的行为

| 模式 | 卡片数量 | 更新频率 | 可操作性 |
|------|----------|----------|----------|
| 巡航 | 3–5张（今日最重要） | 每30分钟或事件触发 | 可点击展开、派单 |
| 验证 | 自动插入1张（当前事件） | 实时 | 可确认/忽略 |
| 事件 | 1张（当前事件命题） | 实时刷新 | 可升级/排除 |

## 组件 6：消防安全健康度

**简介：** 将全楼消防状态压缩为一个综合评分，并支持 360° 一键检测与钻取。

### 评分维度

总分 100 分，由四个维度加权计算：

| 维度 | 权重 | 子指标示例 |
|------|------|------------|
| 物的不安全状态 | 40% | 设备在线率、设备寿命健康度、消防水压达标率 |
| 管理的遗漏项 | 30% | 巡检完成率、隐患整改及时率、年检合规率 |
| 环境不稳定因素 | 20% | 高风险区域占用情况、电气异常监测值、气象风险 |
| 人的不安全行为 | 10% | 消防通道违规占用、禁烟区违规识别（视频AI） |

### 360° 一键检测

触发后，系统在 30 秒内完成全楼扫描，输出：
- 各维度当前得分 + 与上次检测对比变化
- Top 3 失分项（含命题卡）
- 改善建议（按整改难度从低到高排序）
- 预计整改后的理论得分提升

### 各模式下的状态

| 模式 | 显示位置 | 显示内容 |
|------|----------|----------|
| 巡航 | Zone L 顶部 | 综合分数 + 7日趋势折线 |
| 验证 | Zone L 顶部（实时微调） | 当前事件对评分的影响动态显示 |
| 事件 | Zone L 收起 | 仅显示分数，不展开 |
| 报警/熔断 | **完全隐藏** | — |
| 办公 | Zone C 核心区域 | 四维度雷达图 + 历史趋势 + 360°检测触发 |

## 新增组件 A：模式状态机控制器

**简介：** 管理八种模式之间的切换逻辑，是整个系统的"操作系统控制层"。

### 功能说明

**自动切换规则引擎**：
- 基于传感器信号、AI置信度阈值、时间因素，自动触发模式切换
- 每次模式切换必须有明确的触发依据（记录在系统日志）
- 同一事件不重复触发（防抖机制：同一信号源 10 秒内只触发一次）

**人工干预机制**：
- 顶部状态栏 → 模式选择器（始终可见，点击展开）
- 人工切换需选择原因（防止误操作）：`原因：巡逻确认误报` / `原因：演习测试`
- 报警模式下的切换需双重确认

**越级规则（跳过中间模式）**：
- 已确认火情信号直接进入报警模式（不经过验证/事件模式）
- 预设紧急联系人可远程强制触发任意模式（有审计日志）

**模式切换动画规范**：
- 巡航→验证：无感切换（Zone L 局部更新，无全屏过渡）
- 验证→事件：Zone C 轻微放大，摄像头视图渐入
- 事件→报警/熔断：全屏黑色遮罩 0.5秒 → 报警界面闪出（强视觉冲击，确保值班员不会错过）
- 报警→巡航（解除）：红色边框渐退，3D模型慢速复原，Zone L 渐显

**节点类型**：

## 新增组件 E：合规预检模块

**简介：** 年检前 AI 自动做差距分析，将合规压力转化为可执行的整改清单。与方法论中"合规预检模式"对应。

**4. 预检报告生成**
- 模拟年检官验收视角，生成预检自评报告
- 标注高风险项（年检不通过概率 > 60%）
- 支持自然语言指令：`"生成本楼10月消防合规自评报告"`

| 巡航 | 后台运行，数据持续更新 |

# 五、待补充与开放问题

> 这些问题需要在下一轮讨论中确认，影响 Demo 设计和产品定义。

1. **多角色权限**：值班员 vs 消防主管 vs 消防指挥员，三个角色在同一模式下的可见内容和可操作权限是否有差异？
2. **移动端延伸**：报警模式下消防主管的手机端接收什么内容？是镜像大屏还是精简版指挥端？
3. **BIM 数据新鲜度**：建筑改造后 BIM 模型多久更新一次？模型与实际不符时如何降级展示？
4. **L4 预授权的法律边界**：自动关防火门、解锁门禁是否需要物业和业主的书面授权？需对接法律合规咨询。
5. **视频 AI 模型部署方式**：本地部署还是云端推理？直接影响判定延迟和隐私合规。
6. **公共模式的业主协议**共大屏播放逃生信息时，商场：公广告内容须协议约定可被强制中断，需要商务条款支持。
