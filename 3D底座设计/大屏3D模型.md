# 智慧消防大屏 · 中央模型区（Zone C）设计说明

> 版本 v0.1 | 中央模型区是整个驾驶舱的核心视图，承载所有空间化信息的可视化表达与交互入口。本文档完整定义其图层体系、视觉规范、交互行为与模式适配规则。

---

## 一、设计哲学

中央模型区的本质是**空间化的认知接口**。

传统大屏的中央区域是"装饰性的"——一个旋转的3D建筑，功能是"看起来很炫"。AI 驾驶舱的中央区域是"功能性的"——它是整栋建筑所有空间信息的**唯一可信视图**，也是所有操作动作的**空间锚点**。

三条设计原则支撑这个定位：

**原则一：空间即语言**
所有信息优先用空间关系表达，而不是表格或列表。"3楼东北角探测器离线"在大屏上应该是一个发光的异常点，而不是一行文字。用户的眼睛能在3秒内定位到它。

**原则二：图层即视角**
不同角色、不同模式需要看到不同的空间事实。图层不是"叠加显示的功能开关"，而是**针对不同决策视角的信息过滤器**。值班员需要的图层和消防指挥员需要的图层，在内容和密度上都应该不同。

**原则三：悬停即命题**
鼠标移入任何空间对象（设备、区域、路径、人员），必须能即时触发一张**命题卡**，而不只是一个"设备ID：S3-NE-047"的提示框。这是方法论中"命题优先于指标"原则在空间视图里的直接体现。

## 二、基础视图体系

中央模型区支持三种基础视图模式，可手动切换，部分模式下自动触发：

### 2.1 标准 3D 视图（Standard 3D）

**适用模式**：巡航、验证、事件、报警

**视觉特征**：
- 建筑完整呈现，外立面半透明（透明度约 30%），内部楼层可见
- 默认视角：建筑偏右前方 45° 俯视，略带仰角（约 25°），使建筑有纵深感但不遮挡顶层
- 楼层间有细线分隔，强调楼层单元的独立性
- 地面投影：建筑在地面产生淡色轮廓投影，增强空间感

**色调基础**：
- 正常态：深蓝灰色调为主（#0D1B2A 系），建筑白模略带冷光
- 告警态：不改变全局色调，仅异常区域局部色彩饱和度提升
- 避免：任何"全屏变红"的粗暴视觉反馈（红色只用于精确标注，不用于全局氛围渲染）

**相机行为**：
- 鼠标滚轮：缩放（楼层聚焦）
- 左键拖动：旋转（自由视角）
- 右键拖动：平移
- 双击楼层：相机飞向该楼层，展开楼层平面视图
- 双击建筑外部空白处：复原默认视角
- 报警/熔断态：相机锁定起火点，旋转禁用（可双击临时解锁 5 秒）

---

### 2.2 楼层展开视图（Floor Explode）

**适用模式**：巡航（楼层详情查看）、办公

**触发方式**：双击任意楼层；或左侧楼层列表点击

**视觉特征**：
- 选中楼层展开放大至中央，其余楼层收缩为薄片并向上下各自推开（爆炸图模式）
- 选中楼层以平面视角呈现，墙体/房间/通道清晰可见
- 楼层内所有设备点位在此视图下密度最高，全部可见
- 顶部显示：楼层号 + 建筑面积 + 当前人员数量 + 消防分区数量

**返回方式**：
- 点击楼层外空白区域：收起该楼层，恢复全楼视图
- ESC 键：快速返回默认视角

---

### 2.3 X-Ray 透视视图（X-Ray）

**适用模式**：应急指挥、推演/接管

**触发方式**：模式切换时自动激活；或手动按钮切换

**视觉特征**：
- 外立面完全透明（透明度 100%），仅保留楼板和承重墙的线框轮廓
- 建筑内部空间完全可见，消防管网、电缆桥架等隐蔽层可叠加显示
- 整体色调：深色背景 + 蓝绿色线框（参考医学影像风格，传达"透视/诊断"的认知隐喻）
- 结构线框线宽：楼板 1.5px，墙体 1px，门窗洞口 0.5px（通透感）

**特殊功能**：
- 可按"结构类型"过滤：只看防火墙、只看逃生通道、只看消防设施
- 横截面切割：拖动水平切割面，查看任意标高的截面图

## 三、图层体系完整规范

图层是中央模型区的核心架构。以下定义 12 个图层的完整规格：

---

### L-0 建筑基础白模层

**图层属性**

| 项目 | 规格 |
|------|------|
| 数据来源 | BIM模型（IFC格式，精度 LOD300） |
| 更新频率 | 静态；建筑改造后人工更新，变更日志可查 |
| 默认状态 | 始终开启，不可关闭 |
| 所有模式 | 显示 |

**视觉规范**

- 外立面：半透明白色（opacity 0.35），带细线网格纹（网格间距约 3m 对应 BIM 分格）
- 楼板：略深白色（opacity 0.55），强调楼层分隔
- 承重柱/墙：不透明白色（opacity 0.85），作为空间坐标骨架
- 门洞/窗洞：开口处透明，形成空间通透感
- X-Ray 模式下：所有面替换为线框（颜色 #1A4B6B，线宽如上节规范）

**鼠标交互**

- 悬停楼板/楼层区域：显示楼层基础信息卡（详见第四节）
- 点击外立面：无响应（避免误触）
- 双击楼层：触发楼层展开视图

**BIM 关联属性**（点击后可查询）
- 建设年份、结构形式、耐火等级
- 各楼层：建筑面积、功能用途分区、最大人员容量
- 各防火分区：面积、防火门数量、最近一次检验日期

---

### L-1 消防探测设备层（感知网络）

**图层属性**

| 项目 | 规格 |
|------|------|
| 数据来源 | 消防主机（实时）+ BIM设备台账（静态属性） |
| 更新频率 | 实时（WebSocket，延迟 < 2秒） |
| 默认状态 | 开启 |
| 适用模式 | 全部 |

**设备类型与图标规范**

所有设备以统一的**三层结构**呈现：底层图标（设备类型）+ 状态光环（运行状态）+ 数值气泡（关键读数，可选）

| 设备类型 | 图标形状 | 正常态 | 告警态 | 离线态 | 故障态 |
|----------|----------|--------|--------|--------|--------|
| 点型感烟探测器 | 实心圆 Ø8px | 白色，静止 | 红色，脉冲闪烁（2Hz） | 灰色，X标记 | 橙色，叹号 |
| 点型感温探测器 | 空心圆 Ø8px | 白色，静止 | 橙红色，脉冲 | 灰色，X标记 | 橙色，叹号 |
| 线型光束感烟 | 虚线段（两端节点） | 白色 | 红色线段 | 灰色 | 橙色 |
| 火焰探测器 | 火焰图标 12px | 白色 | 红色，跳动动效 | 灰色 | 橙色 |
| 可燃气体探测器 | 六边形 10px | 白色 | 黄色，脉冲 | 灰色 | 橙色 |
| 电气火灾监控 | 闪电图标 12px | 白色 | 黄色 | 灰色 | 橙色 |
| 手动报警按钮 | 方形 10px | 白色 | 红色实心 | 灰色 | 橙色 |
| 消防广播模块 | 喇叭图标 12px | 白色 | 激活态（蓝色） | 灰色 | 橙色 |

**状态光环规范**
- 正常：无光环（保持视觉安静）
- 预警：淡黄色光环，慢速呼吸（1Hz）
- 告警：红色光环，快速脉冲（3Hz）+ 光圈向外扩散动效
- 离线：灰色，静止，无光环，图标中心显示"×"
- 故障：橙色光环，中速脉冲（2Hz）

**密度管理**（避免密集设备导致视觉混乱）
- 全楼视图（缩放级别 1）：仅显示有状态变化（告警/离线/故障）的设备；正常设备以楼层聚合徽章表示（如"3F · 正常 ✓ 47台"）
- 楼层视图（缩放级别 2，双击楼层后）：当前楼层所有设备全部显示
- 设备详情（缩放级别 3，鼠标滚轮放大）：设备图标放大，显示设备编号和最新读数

**鼠标悬停响应**（详见第四节悬停卡片规范）
→ 触发：探测设备命题卡

### L-2 消防系统设施层（灭火网络）

**图层属性**

| 项目 | 规格 |
|------|------|
| 数据来源 | 消防联动控制器（实时）+ BIM机电图层 |
| 更新频率 | 实时（< 2秒）|
| 默认状态 | 开启 |
| 适用模式 | 全部（报警模式下重点显示） |

**设施类型与视觉规范**

| 设施类型 | 图标 | 正常态 | 激活/动作态 | 故障态 |
|----------|------|--------|-------------|--------|
| 自动喷淋头 | 十字形 + 中心点 | 蓝色，静止 | 蓝色，水滴下落动效 | 橙色 |
| 消防栓（室内） | 竖矩形 + 圆圈 | 蓝色 | 蓝色高亮 | 橙色 |
| 消火栓箱 | 方形带圆点 | 蓝色 | — | 橙色 |
| 防火门 | 门框图标 | 绿色（开启）/ 白色（关闭） | 动态开关动画 | 橙色 |
| 防火卷帘 | 竖向矩形 | 绿色（收起）/ 红色（展开） | 展开动画 | 橙色 |
| 消防水泵 | 泵图标（圆+叶片） | 白色 | 绿色高亮（运行中）| 橙色 |
| 排烟风阀 | 风叶图标 | 白色（关闭） | 蓝绿色旋转（开启） | 橙色 |
| 排烟风机 | 大型风叶 | 白色 | 蓝绿色旋转 | 橙色 |
| 气体灭火装置 | 瓶罐图标 | 白色 | 红色（释放中） | 橙色 |

**消防水管网**（特殊规范）
- 以细线（1px）连接所有喷淋头，形成管网拓扑
- 正常态：深蓝色（#1E4B7A），低透明度（opacity 0.4），避免视觉噪声
- 激活区段：管段变为亮蓝色（#4A90D9），透明度提升（opacity 0.8）
- 管网压力可视化：悬停任意管段，显示当前水压（MPa）

**联动状态联动规范**
- 报警模式触发后，已执行联动的设施：整体高亮 + 状态标签（"已启动 14:32"）
- 未执行的联动建议：黄色边框 + 倒计时标签
- 执行中：白色呼吸光晕

**鼠标悬停响应**
→ 触发：消防设施命题卡（含实时状态 + 最近维保记录 + 联动配置）

### L-3 视频监控点位层

**图层属性**

| 项目 | 规格 |
|------|------|
| 数据来源 | 视频管理系统（VMS）+ BIM摄像头台账 |
| 更新频率 | 摄像头状态实时；视频流按需调取 |
| 默认状态 | 关闭（避免初始视觉过载）|
| 适用模式 | 事件、报警（自动开启）；其他模式手动 |

**视觉规范**

摄像头以**三要素**呈现：点位图标 + 朝向箭头 + 视锥范围

| 要素 | 规范 |
|------|------|
| 点位图标 | 摄像头图标，正常态白色，离线灰色，AI分析中蓝色闪烁 |
| 朝向箭头 | 从图标延伸的细箭头，指向摄像头朝向（水平投影） |
| 视锥范围 | 以摄像头为顶点的扇形区域（角度对应实际镜头FOV），正常态极低透明度（opacity 0.08），悬停时提升至（opacity 0.25） |
| PTZ标识 | 可旋转摄像头图标添加环形箭头，悬停时显示PTZ控制入口 |

**AI分析状态**
- 待机：无特殊标记
- 正在分析（事件模式触发）：图标蓝色脉冲 + "AI分析中"标签
- 分析完成（低风险）：绿色✓ + "误报 82%"标签
- 分析完成（疑似火情）：红色 ! + "火情 71%"标签

**关联摄像头高亮逻辑**
- 当某传感器触发时，覆盖该传感器位置的最近摄像头（1–3个）自动高亮（蓝色边框）
- 其余摄像头透明度降低（opacity 0.15），视觉降噪

**鼠标悬停响应**
→ 触发：摄像头信息卡 + 实时画面缩略图（详见第四节）

### L-4 人员热力与定位层

**图层属性**

| 项目 | 规格 |
|------|------|
| 数据来源 | Wi-Fi探针 / 门禁刷卡系统 / 人员定位系统（UWB）|
| 更新频率 | 30秒刷新（Wi-Fi探针精度）；UWB时实时 |
| 默认状态 | 关闭（正常模式）；报警/应急模式自动开启 |
| 适用模式 | 应急指挥、推演（核心）；其他模式按需 |
| 隐私说明 | 平时态不显示个人身份信息，仅显示人数/热力；战时态可关联门禁信息追踪特定人员 |

**双模式呈现**

**模式A：聚合热力图**（默认，用于正常监控）
- 各楼层叠加热力渐变色（冷→热：#1E3A5F→#E24B4A）
- 颜色对应人员密度（人/㎡）
- 正常人流不触发任何告警
- 密度超阈值（人员密度 > 设计容量的 85%）→ 热力图高亮 + 轻微预警卡片

**模式B：个体光点图**（应急模式，战时态）
- 每个被探测到的人员显示为一个光点
- 光点颜色分类：
  - 绿色 ：已疏散至安全区
  - 蓝色 ：移动中（正在疏散）
  - 黄色 ：静止超过 60 秒（可能被困）
  - 红色 ：静止超过 3 分钟（高风险被困）
  - 白色 ：消防/救援人员（门禁标签标记）
- 悬停光点：显示探测来源（哪个Wi-Fi AP / 门禁节点）、最后活动时间、所在房间（BIM关联）

**聚合统计显示**
- 每个楼层自动显示人数标签（右上角徽章）
- 起火楼层：被困人员数量大字显示，红色高亮

### L-5 逃生路径层

**图层属性**

| 项目 | 规格 |
|------|------|
| 数据来源 | AI实时路径计算（基于BIM拓扑 + 传感器状态）|
| 更新频率 | 每 30 秒重新计算；传感器状态变化时立即重算 |
| 默认状态 | 关闭；报警模式自动开启 |
| 适用模式 | 报警、应急指挥、推演、公共（核心） |

**路径状态类型**

| 状态 | 颜色 | 线型 | 说明 |
|------|------|------|------|
| 推荐路径（最优） | 绿色 #4CAF50 | 实线，粗（3px），流动箭头动效 | AI计算的当前最优逃生路径 |
| 备选路径 | 蓝色 #2196F3 | 实线，中等（1.5px），静止箭头 | 次优路径，备用 |
| 危险/封闭路径 | 红色 #E24B4A | 虚线，带 X 标记 | 已被烟气/火情封锁 |
| 监测中路径 | 黄色 #FFC107 | 点划线 | 当前状态不确定，谨慎通行 |
| 疏散出口 | 绿色实心矩形，大箭头 | — | 出口位置 + 当前状态（可用/封闭/拥堵）|

**流动箭头动效规范**
- 推荐路径上的箭头沿路径方向流动（类似水流）
- 流速：1.5m/s视觉等效（与人员行走速度相近，直觉引导）
- 路径分叉点：箭头放大并停顿 0.5 秒，强调"方向选择点"

**动态避让逻辑**
- 某条路径上的传感器触发 → 路径状态变为"监测中"（黄色）
- 视频 AI 确认该区域有烟火 → 路径状态变为"封闭"（红色）→ AI 立即重算并更新推荐路径
- 某出口人员密度过高 → 出口标注"拥堵预警" + AI 建议分流至次出口

**楼层间路径**
- 楼梯：路径在楼梯位置以垂直箭头（↓）连接上下楼层
- 逃生电梯：仅在消防队确认可用时显示（默认隐藏，防止误导）

**鼠标悬停响应（路径段）**
→ 触发：路径信息卡（路径长度、预计疏散时间、当前通行状态）

### L-6 烟气蔓延推演层

**图层属性**

| 项目 | 规格 |
|------|------|
| 数据来源 | AI蔓延模型（输入：起火点位置 + 风阀状态 + 建筑通风拓扑 + 室内气流） |
| 更新频率 | 每 60 秒重新计算（计算量较大）|
| 默认状态 | 关闭；应急指挥/推演模式手动开启或自动激活 |
| 适用模式 | 应急指挥、推演（核心）|
| 数据性质 | 预测模型输出，非实测，带置信区间 |

**时间维度**
- 推演图层绑定时间轴（Zone B 底部时间控制器）
- 初始状态：t=0（火情确认时刻）
- 可拖动至：t+1min、t+3min、t+5min、t+10min
- 每个时间节点独立渲染

**烟气渲染规范**

| 要素 | 规范 |
|------|------|
| 烟气体积 | 半透明体积雾效，颜色 #3D2B1F（深灰棕），随时间扩散 |
| 密度梯度 | 核心区：opacity 0.75；扩散边缘：opacity 0.15–0.35，渐变过渡 |
| 危险边界线 | 橙红色等高线，标注"当前预测边界" |
| 置信带 | 危险边界线外侧淡黄色区域，表示模型不确定性范围 |
| 垂直分层 | 烟气先聚集在天花板层，再向下填充（符合物理规律）；视图中以透明度分层表达 |

**交互控制**
- 时间轴拖拽：查看 t 时刻的烟气状态
- "重置推演"按钮：清除当前推演，重新从起火点计算
- "修改排烟状态"：用户可假设打开/关闭某个风阀，系统重新推演（对应反事实推演模式）
- 与逃生路径层联动：烟气到达某区域时，对应逃生路径自动变为红色封闭

**数据标注**
- 每次推演结果标注：计算时间、使用的模型版本、置信度
- 明显标识"AI推演，非实测"（避免指挥员误认为是实时传感器数据）

## 五、悬停命题卡完整规范

**核心原则：鼠标悬停任何空间对象，弹出的不是"工具提示"，而是"命题卡"。**

### 5.1 通用卡片交互规则

**触发**：
- 鼠标悬停超过 0.8 秒（防止移动过程中误触发）
- 触控屏：长按 0.5 秒

**位置**：
- 卡片出现在鼠标指针右上方（距离指针 12px）
- 当卡片超出屏幕边缘时自动镜像翻转

**尺寸**：
- 摘要态：宽 240px，高度自适应（通常 120–180px）
- 展开态：宽 360px，高度自适应（完整四要素）

**动效**：
- 出现：向上淡入 + 轻微上移（100ms，ease-out）
- 消失：淡出（80ms）
- 摘要→展开：宽度展开 + 内容渐入（150ms）

**卡片内固定元素**：
- 左上：图层颜色标识圆点 + 对象类型标签（如"感烟探测器"）
- 右上：AI置信度（如有推断）
- 右上：授权级别标识（如有对应建议）
- 底部：行动按钮区（最多3个，过多则折叠）
- 底部：展开/收起按钮（摘要态 ↔ 展开态）

## 七、性能与工程约束

中央模型区的视觉质量最终受限于工程实现，以下约束需在产品设计阶段明确。

### 7.1 渲染性能基线

| 指标 | 最低要求 | 推荐要求 |
|------|----------|----------|
| 帧率 | 30 FPS（全图层开启时） | 60 FPS |
| 模型多边形数 | < 500万（全楼） | < 200万（LOD优化后）|
| 图层切换响应 | < 200ms | < 100ms |
| 相机动画（飞向楼层）| < 1.5秒 | < 0.8秒 |
| 悬停卡片出现延迟 | < 300ms | < 150ms |

### 7.2 LOD（细节层次）策略

为保证大范围视图的流畅性，采用三级 LOD：

| 级别 | 触发条件 | 模型精度 | 图标密度 |
|------|----------|----------|----------|
| LOD-1（全楼俯视）| 相机距离 > 50m | 建筑外轮廓 + 楼层分割 | 仅显示告警设备 |
| LOD-2（楼层视图）| 相机距离 10–50m | 完整室内结构 | 全部设备可见 |
| LOD-3（设备详情）| 相机距离 < 10m | 高精度构件 | 设备标签全显示 |

### 7.3 BIM 模型更新机制

- 建筑改造后，物业管理员在系统内导入新版 BIM 文件（支持 IFC 2x3 / IFC 4）
- 更新前：新旧版本对比展示，差异构件高亮（蓝色=新增，橙色=变更，灰色=删除）
- 更新后：系统自动重新关联所有设备的空间位置（基于设备编号匹配，人工确认异常）
- 模型与实际不符时的降级策略：
  - 标注"BIM版本 xxxx，最后更新 xx天前，与实际可能有偏差"
  - 在卡片和推演结果中显示"位置精度降级"警告

### 7.4 数据接入优先级

并非所有图层的数据都能实时获取，按优先级分级：

| 优先级 | 图层 | 数据性质 | 断线降级处理 |
|--------|------|----------|-------------|
| P0（不可缺失）| L-0, L-1, L-2 | 实时 | 离线缓存最近5分钟，显示"数据可能延迟" |
| P1（重要）| L-3, L-4, L-5 | 实时 | 使用最近一次数据+时间戳提示 |
| P2（辅助）| L-6, L-7, L-8, L-9 | 实时/准实时 | 灰显，提示"数据获取中" |
| P3（参考）| L-10, L-11, L-12 | 静态/历史 | 正常显示，不影响核心功能 |

| 方法论条目 | Zone C 实现 |
|------------|-------------|
| 命题优先于指标 | 悬停卡片强制四要素结构，禁止纯数字展示 |

| 四重时间观 | 时间轴绑定烟气推演层；历史事件层提供"历史"维度；设备寿命预测提供"近未来"维度 |
